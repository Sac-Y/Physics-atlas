# 教训沉淀

## 2026-07-02 · 自动播放的等待期必须有可见的进度与手动控件（用户报"导览没反馈"）

**发生了什么**：导览每站自动停留 10~12 秒，期间没有进度指示、没有手动前进控件；功能完全正常，但用户体感是"点了没反应、像死了"。而点击星空会静默退出导览，被误读为 bug。

**规则**：任何自动推进的序列（导览/幻灯/轮播）必须同时给三样东西：看得见的进度（倒计时线/点）、手动控制（上一步/下一步）、明确的退出反馈。"功能在跑"和"用户知道它在跑"是两件事，后者才算做完。

## 2026-07-02 · 全屏 WebGL 性能要在第一帧就设防（用户报整机卡顿后才修）

**发生了什么**：星空场景按视觉优先一路堆料（2x 像素比、每支星云 381 张最大 1500px 加法混合大贴图、全分辨率 bloom、无效 MSAA），用户使用时 GPU 填充率被打满，拖垮系统合成器，所有页面一起卡。

**规则**：
1. 全屏 WebGL 项目在写 renderer 的第一行就定像素比上限（柔光场景 1.5 足够），全部 shader 的 uPixelRatio 与 renderer 同源（集中到一个 display.js 常量）。
2. 加法混合的大 point sprite 是 overdraw 之王：粒子数×单张尺寸×每像素算力三者乘积要预算化；"数量减、单张透明度补偿"可以在总亮度不变下大幅降载。
3. 皮壳像素（mask≈0 区域）先 discard 再跑贵计算（fbm 等）。
4. 用 EffectComposer 时 antialias:true 无效且浪费；bloom 用半分辨率目标。
5. 视觉迭代期每轮该看一眼 GPU 负载，而不是等用户报卡。

## 2026-07-02 · 后台标签页 rAF 暂停，影响一切基于预览的验证

**规则**：预览标签页在后台时 requestAnimationFrame 完全暂停——动画不走、帧率测不了、模拟滚轮"无效"。解法：给应用加受控步进接口（`__atlas.step(n)` / `setTargetRadius`），验证子代理用它驱动画面；真实帧率必须前台标签页测。

## 2026-07-02 · 运行时标签避让不能在相机运动中重算

**规则**：基于 getBoundingClientRect 的标签避让，在相机运动中拿到的是移动快照，会成片误杀标签（配合 CSS 过渡表现为"缩放时标签全消失，停稳几秒才回来"）。相机运动时冻结避让状态，停稳后再重算。

## 2026-07-03 · 排队给子代理的消息会在其停摆时丢失(轴文字任务凭空消失)

**发生了什么**:向正在运行的子代理 SendMessage 派新任务,消息进了"下个工具回合投递"的队列;子代理随后停摆,该消息随之丢失。重启后它对这个任务毫无记忆,还合理地质疑协调器"串线"。双方各自视角都自洽,浪费了一整轮往返。

**规则**:
1. 给运行中的子代理排队派单后,若它在回报前停摆过,**重启时必须把未确认的任务清单完整重述**,不能假设队列消息已送达。
2. 子代理回报里没提到的在途任务 = 大概率没收到,先核对再指责返工。
3. 关键任务派单后,以它的回报里"复述了这个任务"为送达凭证。
